# tc-running

## Purpose

`tc-running` 负责执行 pytest 自动化测试脚本，并生成测试执行报告。

## When to use

仅在以下场景使用：

```text
完整模式执行闭环
用户明确要求执行测试并生成报告
```

`tc-running` 不是设计阶段默认步骤。

## Preconditions

使用前必须确认：

1. pytest 项目路径已明确。
2. 测试环境配置已明确。
3. 执行参数已明确或可以使用默认参数。
4. 用户确认可以执行测试。

## Inputs

```text
pytest 项目路径
测试环境配置
执行参数，可选
```

## Outputs

```text
test-design/reports/test_report.md
```

## Required report sections

`test_report.md` 应包含：

1. 执行环境。
2. 执行时间。
3. 总用例数。
4. 通过数。
5. 失败数。
6. 跳过数。
7. 失败用例明细。
8. 失败原因分析。
9. 阻塞问题。
10. 后续建议。

## Execution rules

1. 不要在环境配置缺失时强行执行。
2. 不要跳过失败或隐藏失败原因。
3. 测试失败时，应区分真实缺陷、环境问题、数据问题、脚本问题。
4. 报告必须保留失败详情和后续建议。
5. 如果执行会影响共享环境或真实数据，应先提醒并等待确认。
6. 执行结果不能直接作为最终准出结论，必须经过人工评审。

## Review gate

```text
Gate：执行结果评审
```

确认内容：

1. 失败用例是否为真实缺陷。
2. 环境和数据问题是否已区分。
3. 阻塞问题是否已记录。
4. 是否满足准出条件。
5. 是否需要回流更新用例或脚本。

## Next step

测试执行闭环结束。如需求、接口或业务规则变更，应按变更回流机制更新上游产物。
