# 测试设计规格：门店活动报名营销工具

## 需求概要

本需求是在快速营销系统中新增可复用的「门店活动报名」营销工具，首期活动示例为“赘肉换牛肉 / 卷腹换好虾”。

核心链路：

1. 运营在营销后台配置活动、门店范围、活动说明、页面装修和活动属性。
2. 顾客通过小程序 H5 进入活动页。
3. 顾客完成会员鉴权后选择门店、选择男子组 / 女子组并报名。
4. 顾客在活动挑战期内提交成绩。
5. 成绩进入待审核状态。
6. 门店员工在 YHDOS 审核通过或驳回。
7. 审核通过成绩进入门店 + 性别分组排行榜。
8. 顾客查看排行榜、全部排名和我的记录。
9. 运营在活动详细数据页查询、筛选并导出活动数据。

## 复杂度评估

结论：复杂模式。

原因：

- 功能模块数 > 3：C 端 H5、YHDOS 审核端、营销后台配置、活动详细数据、排行榜和排名口径、权限和数据约束。
- 涉及多角色：顾客、门店员工、运营。
- 涉及多端联动：C 端报名 / 提交、YHDOS 审核、C 端榜单刷新、后台数据查询 / 导出。
- 测试场景为 Web 应用测试。
- 需要验证数据状态和排名口径一致性。

## 测试范围

### 包含

#### 1. 营销后台配置

- 活动名称。
- 活动副标题。
- 活动时间。
- 报名截止时间。
- 活动门店范围。
- 活动说明。
- 活动头图。
- 主题文案。
- 活动属性：挑战资格、挑战时长、奖励范围。
- 配置必填校验。
- 报名截止时间与活动时间范围校验。
- 活动门店至少选择 1 家。

#### 2. C 端 H5

- 活动页展示。
- 活动状态展示：未开始、进行中、报名截止、已结束。
- 当前门店展示和未报名前门店切换。
- 报名流程。
- 男子组 / 女子组选择。
- 已报名状态展示。
- 成绩提交。
- 每人每天 1 次未驳回成绩限制。
- 门店排行榜 Top5。
- 查看全部排名。
- 我的记录。
- 参与说明。
- 按 B 版本原型验证，A 版本不作为交付入口。

#### 3. YHDOS 审核端

- 员工登录态和门店权限。
- 当前员工授权门店展示。
- 待审核 / 已通过 / 已驳回 / 全部 Tab。
- 手机号 / 姓名搜索。
- 审核卡片字段。
- 审核通过。
- 审核驳回。
- 审核后榜单和个人最好成绩刷新。
- 不提供门店筛选。
- 不展示满 88 元凭证、身份证 / 年龄、健康声明核验项。

#### 4. 活动详细数据页

- 从营销活动列表进入详细数据页。
- 手机号筛选。
- 门店筛选。
- 成绩排序：成绩从高到低、成绩从低到高、参与时间倒序。
- 查询。
- 重置。
- 导出 Excel。
- 导出字段与列表一致。

#### 5. 排行榜和数据口径

- 只统计审核通过成绩。
- 待审核不入榜。
- 已驳回不入榜。
- 按活动 + 门店 + 性别分组计算。
- 同一用户多条审核通过成绩只取最高成绩。
- 个人排名与排行榜使用同一口径。
- 活动详细数据中的排名字段使用同一口径。
- 同分排名采用并列排名。

#### 6. 权限与数据约束

- 顾客只能查看自己的报名和记录。
- 门店员工只能审核授权门店数据。
- 运营可查看详细数据和导出。
- 手机号展示和导出脱敏。
- 首期不存储身份证号、身份证照片、小票照片或线上订单截图。
- 首期不做订单系统自动校验满 88 元资格。
- 首期顾客端不提供换店入口。

### 排除

- 订单系统自动校验满 88 元资格。
- 身份证号、身份证照片、小票照片、线上订单截图存储和校验。
- 跨门店总榜。
- 顾客端换店入口。
- A 版本 C 端页面交付验证。
- 真实图片上传链路深度测试。
- 服务端 CSV 异步导出实现细节。
- 自动化脚本生成和执行。

## 测试策略

选择策略：模块全铺。

### 策略说明

按业务端和能力域拆分模块，分别覆盖：

1. 营销后台配置。
2. C 端 H5。
3. YHDOS 审核端。
4. 活动详细数据页。
5. 排行榜与排名口径。
6. 权限与数据约束。
7. 跨端主链路联动。

### 适用场景

适用于本需求这种多端、多角色、多状态、多数据口径的 Web 应用功能。

### 优点

- 覆盖更均衡。
- 不容易遗漏后台配置、YHDOS 审核和导出能力。
- 便于后续拆分测试计划和用例模块。
- 适合形成可复用测试资产。

### 缺点

- 用例数量会比主链路优先策略更多。
- 需要后续在测试计划中明确 P0 / P1 / P2，避免执行成本过高。
- 如果接口和环境未就绪，部分用例只能先作为设计用例。

## 重点关注区域

用户选择：主链路联动。

重点验证：

1. 后台配置活动后，C 端正确展示活动信息。
2. 顾客报名后，按钮状态和用户绑定门店正确变化。
3. 顾客提交成绩后，成绩进入待审核。
4. YHDOS 审核通过后，成绩进入排行榜。
5. YHDOS 驳回后，成绩不入榜，顾客可重新提交。
6. 同一用户多次通过成绩只取最高分。
7. C 端排行榜、我的记录、后台详细数据排名口径一致。
8. 门店员工只能看到和审核本门店数据。
9. 运营导出字段与详细数据列表一致。
