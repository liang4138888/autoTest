# 测试计划：门店活动报名营销工具

## 关联规格

- 规格文档：`docs/test-cases/specs/2026-05-12-门店活动报名-test-spec.md`

## 模块测试计划

### 模块 1：营销后台活动配置

- **优先级**：P0
- **优先级理由**：活动配置是 C 端展示、报名、提交成绩和数据统计的前置基础，配置错误会导致全链路不可用。
- **模块类型**：UI
- **功能点**：
  - 活动名称、活动副标题、活动说明配置。
  - 活动时间和报名截止时间配置。
  - 报名截止时间必须在活动时间范围内。
  - 活动门店范围选择，至少选择 1 家门店。
  - 活动头图和主题文案配置。
  - 活动属性配置：挑战资格、挑战时长、奖励范围。
  - 必填字段为空时保存失败并提示。
  - 配置保存后 C 端可正确展示对应内容。
- **覆盖维度**：功能测试
- **预估用例数**：18
- **MCP 数据需求**：无
- **依赖**：无

### 模块 2：C 端活动页展示与状态控制

- **优先级**：P0
- **优先级理由**：顾客进入活动后的首要页面，直接影响报名、提交成绩和榜单入口可用性。
- **模块类型**：UI
- **功能点**：
  - 展示活动头图、标题、副标题、活动时间、报名截止时间。
  - 展示活动属性：挑战资格、挑战时长、奖励范围。
  - 展示当前门店。
  - 展示参与说明入口和活动说明弹窗。
  - 活动未开始时不可报名、不可提交成绩。
  - 活动进行中且未超过报名截止时可报名、可提交成绩。
  - 报名截止后不可报名、不可提交成绩，可看榜。
  - 活动结束后不可报名、不可提交成绩，可看历史榜。
  - 仅验证 B 版本原型，A 版本不作为交付入口。
- **覆盖维度**：功能测试
- **预估用例数**：16
- **MCP 数据需求**：无
- **依赖**：模块 1

### 模块 3：C 端门店选择与报名建档

- **优先级**：P0
- **优先级理由**：报名是用户参与活动的核心入口，报名状态决定后续成绩提交、查看全部排名和我的记录。
- **模块类型**：UI
- **功能点**：
  - 未报名前可切换活动门店。
  - 非活动门店不可报名。
  - 点击立即报名打开报名弹窗。
  - 报名弹窗展示当前门店。
  - 报名必须选择男子组或女子组。
  - 报名成功后绑定当前活动和门店。
  - 同一活动下用户只能绑定一家活动门店。
  - 报名成功后立即报名按钮置灰并展示已报名。
  - 未报名时提交成绩按钮置灰。
  - 未报名时查看全部排名置灰。
  - 首期顾客端不提供换店入口。
- **覆盖维度**：功能测试
- **预估用例数**：20
- **MCP 数据需求**：无
- **依赖**：模块 1、模块 2

### 模块 4：C 端成绩提交

- **优先级**：P0
- **优先级理由**：成绩提交是活动参与的核心行为，直接影响审核、排行榜和我的记录。
- **模块类型**：UI
- **功能点**：
  - 已报名用户可点击提交成绩。
  - 未报名用户不可提交成绩。
  - 当前时间不在活动时间内不可提交成绩。
  - 当前时间超过报名截止时间不可提交成绩。
  - 用户当天不存在未被驳回成绩时可提交。
  - 用户当天已有待审核成绩时不可再次提交。
  - 用户当天已有已通过成绩时不可再次提交。
  - 用户当天成绩被驳回后可重新提交。
  - 提交字段包含活动 ID、会员 ID、手机号、门店 ID、性别分组、本次成绩、提交时间。
  - 提交成功后状态为待审核。
  - 待审核成绩不进入排行榜。
- **覆盖维度**：功能测试
- **预估用例数**：22
- **MCP 数据需求**：无
- **依赖**：模块 3

### 模块 5：YHDOS 审核端

- **优先级**：P0
- **优先级理由**：审核结果决定成绩是否入榜，是 C 端提交成绩到排行榜生效的关键节点。
- **模块类型**：UI
- **功能点**：
  - 员工进入 YHDOS 审核页后展示当前授权门店。
  - 页面不提供门店筛选。
  - 待审核、已通过、已驳回、全部 Tab 切换。
  - 按手机号 / 姓名搜索审核记录。
  - 审核卡片展示会员标识、脱敏手机号、性别分组、本次成绩、提交时间、审核状态。
  - 审核卡片不展示满 88 元凭证、身份证 / 年龄、健康声明核验项。
  - 待审核成绩可审核通过。
  - 待审核成绩可审核驳回。
  - 审核通过后记录审核人和审核时间。
  - 审核驳回后记录审核人和审核时间。
  - 审核通过成绩参与榜单。
  - 审核驳回成绩不参与榜单。
- **覆盖维度**：功能测试
- **预估用例数**：24
- **MCP 数据需求**：无
- **依赖**：模块 4

### 模块 6：排行榜与排名口径

- **优先级**：P0
- **优先级理由**：排行榜是活动核心展示结果，且需要与审核状态、个人记录、后台详细数据保持一致。
- **模块类型**：UI
- **功能点**：
  - 默认展示当前门店当前组别 Top5。
  - 支持男子组 / 女子组切换。
  - 未报名时查看全部排名置灰。
  - 报名后可查看完整榜单。
  - 只统计审核通过成绩。
  - 待审核成绩不入榜。
  - 已驳回成绩不入榜。
  - 按活动 + 门店 + 性别分组分别计算。
  - 同一用户多条审核通过成绩只取最高成绩。
  - 同分排名采用并列排名。
  - 个人排名与排行榜口径一致。
  - 活动详细数据中的排名字段与排行榜口径一致。
- **覆盖维度**：功能测试
- **预估用例数**：24
- **MCP 数据需求**：无
- **依赖**：模块 4、模块 5

### 模块 7：我的记录

- **优先级**：P1
- **优先级理由**：我的记录是用户查看个人报名、成绩和排名的核心辅助能力，但不阻断主报名提交链路。
- **模块类型**：UI
- **功能点**：
  - 我的记录入口使用吸底浮层入口。
  - 页面主体不重复展示我的记录。
  - 未报名用户打开我的记录展示未报名状态。
  - 已报名用户展示报名门店和性别分组。
  - 展示历史成绩和审核状态。
  - 展示用户审核通过的最高成绩。
  - 展示用户当前排名。
  - 我的最好成绩只统计审核通过成绩。
  - 待审核和已驳回成绩不作为最好成绩。
- **覆盖维度**：功能测试
- **预估用例数**：14
- **MCP 数据需求**：无
- **依赖**：模块 3、模块 4、模块 5、模块 6

### 模块 8：活动详细数据页与导出

- **优先级**：P1
- **优先级理由**：运营需要通过详细数据页查询活动明细并导出，属于运营闭环能力。
- **模块类型**：UI
- **功能点**：
  - 从营销活动列表进入活动详细数据页。
  - 按手机号筛选。
  - 按门店筛选。
  - 按成绩从高到低排序。
  - 按成绩从低到高排序。
  - 按参与时间倒序排序。
  - 查询结果展示参与时间、手机号、成绩、组别、门店、门店名称、排名、审核状态、审核时间、审核人、会员 ID、报名时间。
  - 手机号按公司规范脱敏展示。
  - 重置筛选条件。
  - 导出 Excel。
  - 导出字段与详细数据列表保持一致。
  - 排名字段使用活动 + 门店 + 组别 + 用户最高审核通过成绩口径。
- **覆盖维度**：功能测试
- **预估用例数**：18
- **MCP 数据需求**：无
- **依赖**：模块 4、模块 5、模块 6

### 模块 9：权限与数据隔离

- **优先级**：P1
- **优先级理由**：涉及顾客、门店员工、运营三类角色，权限错误会造成数据越权或误审核。
- **模块类型**：UI
- **功能点**：
  - 顾客只能查看自己的报名记录。
  - 顾客只能查看自己的历史成绩和当前排名。
  - 顾客不可查看其他用户个人记录。
  - 门店员工只能查看本授权门店审核数据。
  - 门店员工不可审核非授权门店成绩。
  - 运营可查看活动详细数据。
  - 运营可导出活动详细数据。
  - 未登录或未授权用户不能进入对应页面。
  - 手机号在 C 端、YHDOS、后台展示时脱敏。
- **覆盖维度**：功能测试
- **预估用例数**：16
- **MCP 数据需求**：无
- **依赖**：模块 3、模块 4、模块 5、模块 8

### 模块 10：跨端主链路联动

- **优先级**：P0
- **优先级理由**：这是需求的端到端核心闭环，验证后台配置、C 端参与、YHDOS 审核、榜单和后台数据是否一致。
- **模块类型**：UI
- **功能点**：
  - 后台配置活动后，C 端展示活动标题、副标题、头图、活动属性和说明。
  - 顾客报名后，C 端展示已报名并绑定报名门店。
  - 顾客提交成绩后，YHDOS 待审核列表出现该成绩。
  - YHDOS 审核通过后，C 端排行榜出现该用户成绩。
  - YHDOS 审核通过后，我的记录展示最好成绩和当前排名。
  - YHDOS 审核驳回后，C 端排行榜不出现该成绩。
  - YHDOS 审核驳回后，顾客可重新提交符合条件的成绩。
  - 同一用户多次审核通过成绩，只取最高成绩参与排名。
  - 后台活动详细数据展示与 C 端、YHDOS 状态一致。
  - 后台导出结果与详细数据列表一致。
- **覆盖维度**：功能测试
- **预估用例数**：20
- **MCP 数据需求**：无
- **依赖**：模块 1、模块 3、模块 4、模块 5、模块 6、模块 8

### 模块 11：非目标与隐藏能力校验

- **优先级**：P2
- **优先级理由**：用于防止交付范围外能力误展示或误上线。
- **模块类型**：UI
- **功能点**：
  - C 端不展示顾客换店入口。
  - 不做跨门店总榜展示。
  - C 端不展示 A 版本入口。
  - YHDOS 审核卡片不展示身份证号、身份证照片、小票照片、订单截图。
  - YHDOS 审核卡片不展示满 88 元凭证。
  - 首期不触发订单系统自动校验满 88 元资格。
  - 活动配置页不承载活动数据查询能力。
- **覆盖维度**：功能测试
- **预估用例数**：10
- **MCP 数据需求**：无
- **依赖**：模块 1、模块 2、模块 5、模块 8

## 执行顺序

1. 模块 1：营销后台活动配置（P0，无依赖）
2. 模块 2：C 端活动页展示与状态控制（P0，依赖模块 1）
3. 模块 3：C 端门店选择与报名建档（P0，依赖模块 1、2）
4. 模块 4：C 端成绩提交（P0，依赖模块 3）
5. 模块 5：YHDOS 审核端（P0，依赖模块 4）
6. 模块 6：排行榜与排名口径（P0，依赖模块 4、5）
7. 模块 10：跨端主链路联动（P0，依赖模块 1、3、4、5、6、8）
8. 模块 7：我的记录（P1，依赖模块 3、4、5、6）
9. 模块 8：活动详细数据页与导出（P1，依赖模块 4、5、6）
10. 模块 9：权限与数据隔离（P1，依赖模块 3、4、5、8）
11. 模块 11：非目标与隐藏能力校验（P2，依赖模块 1、2、5、8）
